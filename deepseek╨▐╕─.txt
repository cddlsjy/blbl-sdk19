二、修改 dispatchKeyEvent 方法中的方向键处理
2.1 替换 KEYCODE_DPAD_UP 和 KEYCODE_DPAD_DOWN 的处理逻辑
找到原始的 UP / DOWN 处理代码块，大约在 when (keyCode) 内部。原始代码类似：

kotlin
KeyEvent.KEYCODE_DPAD_UP -> {
    if (isSidePanelVisible()) return super.dispatchKeyEvent(event)
    // ... 旧逻辑（显示面板、聚焦等）
}
替换为以下新逻辑（参考 PlayerActivity.kt 第 1695 行附近）：

kotlin
KeyEvent.KEYCODE_DPAD_UP -> {
    if (isSidePanelVisible()) return super.dispatchKeyEvent(event)
    if (osdMode == OsdMode.Full && binding.seekProgress.isFocused) return super.dispatchKeyEvent(event)
    if (osdMode == OsdMode.Full && (binding.topBar.hasFocus() || binding.bottomBar.hasFocus())) return super.dispatchKeyEvent(event)
    playPrevByPlaybackMode(userInitiated = true)
    return true
}

KeyEvent.KEYCODE_DPAD_DOWN -> {
    if (isSidePanelVisible()) return super.dispatchKeyEvent(event)
    if (osdMode == OsdMode.Full && binding.seekProgress.isFocused) return super.dispatchKeyEvent(event)
    if (osdMode == OsdMode.Full && (binding.topBar.hasFocus() || binding.bottomBar.hasFocus())) return super.dispatchKeyEvent(event)
    playNextByPlaybackMode(userInitiated = true)
    return true
}
注意：如果您的代码中没有 osdMode == OsdMode.Full 条件，可直接使用上述逻辑。osdMode 枚举是已有的。

三、在 initControls 方法中添加触摸手势切换视频
找到 initControls 方法（一般在 onCreate 中调用）。在该方法内部，现有代码的最后部分（即所有 setOnClickListener 和 setOnSeekBarChangeListener 之后）添加以下手势检测代码。

3.1 添加手势检测器与触摸监听器
将以下代码块复制到 initControls 的末尾（紧接在 setControlsVisible(false) 和 startProgressLoop() 之前）：

kotlin
val swipeThreshold = ViewConfiguration.get(this).scaledTouchSlop * 4
val swipeVelocityThreshold = ViewConfiguration.get(this).scaledMinimumFlingVelocity * 2
val detector = GestureDetector(this, object : GestureDetector.SimpleOnGestureListener() {
    override fun onDown(e: MotionEvent): Boolean = true

    override fun onFling(
        e1: MotionEvent?,
        e2: MotionEvent,
        velocityX: Float,
        velocityY: Float
    ): Boolean {
        if (isSidePanelVisible()) return false
        if (e1 == null) return false
        val dy = e1.y - e2.y
        val dx = e1.x - e2.x
        if (abs(dy) > abs(dx) && abs(dy) > swipeThreshold && abs(velocityY) > swipeVelocityThreshold) {
            val reversed = BiliClient.prefs.playerGestureDirectionReversed
            val isUp = dy > 0
            val shouldPrev = if (reversed) !isUp else isUp
            if (shouldPrev) {
                playPrevByPlaybackMode(userInitiated = true)
            } else {
                playNextByPlaybackMode(userInitiated = true)
            }
            return true
        }
        return false
    }

    // 可选：保留原有的双击步进逻辑（如果有需要可自行保留，这里仅添加上下滑动切换）
})

val touchListener = View.OnTouchListener { v, event ->
    val handled = detector.onTouchEvent(event)
    if (event.action == MotionEvent.ACTION_UP && handled) v.performClick()
    handled
}
binding.playerView.setOnTouchListener(touchListener)
binding.ijkAspect.setOnTouchListener(touchListener)
注意：BiliClient.prefs.playerGestureDirectionReversed 是您设置中的“手势方向反向”开关，如果您的旧版本没有这个配置项，可以改为固定逻辑（例如 isUp 对应下一首）。但为了兼容性，建议先检查 AppPrefs 中是否存在该字段，若没有则直接使用 isUp 决定切换方向。

3.2 确保 playPrevByPlaybackMode 和 playNextByPlaybackMode 存在
这两个方法在旧版本中应该已经存在（因为它们是播放器切换视频的核心）。如果意外缺失，请从 PlayerActivity.kt 中复制其实现（它们通常位于文件后半部分，处理列表、分P、推荐等）。

五、注意事项
变量/方法存在性检查

osdMode、binding.topBar、binding.bottomBar 等都在旧版本中存在（sdk19.txt 和 sdk21-.txt 中都有定义），可直接使用。

playPrevByPlaybackMode 和 playNextByPlaybackMode 需要确认已实现。如果旧版本中没有，请从 PlayerActivity.kt 中复制这两个方法的完整实现（它们内部会处理列表、分P、推荐等复杂逻辑）。

手势方向反转
如果您的 AppPrefs 中没有 playerGestureDirectionReversed 字段，可以改为简单逻辑：

kotlin
val shouldPrev = dy > 0  // 向上滑动为上一个视频
或者保持原样，因为旧版本的 prefs 可能已包含该字段（检查 AppPrefs 类）。

触摸区域
手势监听设置在 binding.playerView 和 binding.ijkAspect 上，覆盖了视频显示区域和 IJK 渲染区域，与 PlayerActivity.kt 一致。

冲突处理
修改后，原本 UP/DOWN 键用于控制 OSD 显示和聚焦的行为将完全被切换视频取代。这与新版本的行为一致。如果您仍希望保留原有 OSD 控制功能（例如在 OSD 显示时方向键导航），上述代码已经通过 osdMode == Full 条件保留了部分导航能力（当焦点在 SeekBar 或顶栏/底栏时，方向键会继续用于导航，而不是切换视频）。这符合 TV 用户的操作直觉。

六、对 sdk21-.txt 的相同修改
sdk21-.txt 与 sdk19.txt 的结构几乎一致，只是部分 API 调用略有不同（例如使用 OkHttpDataSource、没有系统栏适配等）。但dispatchKeyEvent 和 initControls 的逻辑位置相同，因此您可以完全按照上述步骤对 sdk21-.txt 进行同样的修改。

唯一需要注意的是：sdk21-.txt 中可能没有 binding.ijkAspect（因为 IJK 渲染容器命名不同？）。检查后发现两个文件都有 binding.ijkAspect，所以无影响。

七、验证
修改完成后，编译并运行：

遥控器：按方向键上/下、频道上/下、翻页上/下，应能切换视频列表中的上一个/下一个视频。

触摸屏：在视频画面上快速向上或向下滑动，应能切换视频（可配合手势方向反转设置）。

原有功能：双击边缘步进快进/快退、单击显示/隐藏 OSD 等仍应正常工作。

如果遇到任何问题（例如切换时列表为空或崩溃），请检查 playNextByPlaybackMode / playPrevByPlaybackMode 的实现是否完整，以及 pageListItems 等列表数据是否正确填充。

通过以上修改，您就完全将 PlayerActivity.kt 中的遥控器和触摸手势切换视频功能移植到了旧的 sdk19 和 sdk21- 版本中。如有细节差异，可进一步对照 PlayerActivity.kt 源码进行微调。
